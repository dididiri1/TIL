# [Git] 실무에서 살아남는 Git 완전 정복 가이드 (Stash, Fetch, Reset)

커리어 성장을 준비하며 실무에서 가장 빈번하게 마주치는 Git의 핵심 상황별 대처법과 워크플로우를 정리합니다. 단순히 명령어를 외우는 것이 아니라, 작업 공간을 깨끗하게 유지하고 협업 충돌을 방지하는 '고수의 습관'에 초점을 맞췄습니다.

---




## 1. "아차!" 싶을 때 사용하는 커밋 취소 (Reset)

방금 한 커밋을 취소하거나 메시지를 수정하고 싶을 때, 상황에 맞는 Reset 옵션을 선택하는 것이 중요합니다.

| 옵션 | 커밋 기록 | Index(Stage) | 내 파일 코드 | 주요 용도 |
| :--- | :---: | :---: | :---: | :--- |
| **Soft** | 취소 | **유지** | **유지** | 커밋만 취소하고 바로 다시 커밋할 때 |
| **Mixed** | 취소 | 취소 | **유지** | 작업 내용은 남기고 처음부터 다시 정리할 때 |
| **Hard** | 취소 | 취소 | **삭제** | **모든 걸 커밋 전 상태로 완전히 되돌릴 때** |

> **Tip:** 코드는 괜찮은데 메시지만 오타가 났다면 `Amend` 기능을 활용하세요.

---

## 2. 작업 중간 점검의 핵심, Stash

진행 중인 작업을 잠시 바구니에 담아두는 기능입니다. 특히 **새로 만든 파일(Untracked)**을 처리하는 옵션이 핵심입니다.

### 💡 Include Untracked Files 체크박스란?
Git이 아직 관리하지 않는 '신규 파일'까지 바구니에 넣을지 결정하는 옵션입니다. 실무에서는 작업 환경을 완전히 깨끗하게(Clean) 비우기 위해 **반드시 체크(ON)**하는 것을 권장합니다.

### 💡 Apply vs Pop 차이점
| 기능 | 바구니(Stash List) 상태 | 내 코드 상태 | 특징 |
| :--- | :--- | :--- | :--- |
| **Apply** | **내용 유지** | 코드 복사됨 | 안전하게 적용해보고 나중에 다시 쓸 수 있음 |
| **Pop** | **내용 삭제** | 코드 복사됨 | 작업이 확실히 끝났을 때 목록 정리용으로 사용 |

## 3. 남의 코드 가져오기: Fetch vs Pull

팀원과의 협업에서 가장 중요한 단계입니다. 무턱대고 합치기보다 상태를 먼저 확인하는 습관이 필요합니다.

* **Fetch:** "원격 저장소에 새 소식이 있나?" 확인만 하는 작업 (내 코드 변함 없음)
* **Merge:** 가져온 새 소식을 내 코드와 실제로 합치는 작업
* **Pull:** `Fetch + Merge`를 한 번에 수행하는 작업

> **공식:** `Pull = Fetch (가져오기) + Merge (합치기)`

---

## 4. [실전] 잘못된 Pull 취소하기 (Hard Reset)

팀원의 코드를 `Pull` 받았는데 내 소스와 엉망으로 섞였을 때, `Pull` 하기 직전의 **내 마지막 커밋** 상태로 타임머신을 타는 방법입니다.

1.  **History 뷰**에서 내가 마지막으로 했던 커밋을 찾습니다.
2.  해당 커밋 우클릭 > **Reset > Hard**를 선택합니다.
3.  `Pull`로 들어왔던 남의 코드들이 싹 사라지고 내 커밋 시점으로 완벽히 복구됩니다.

---

## 5. 실무 고수의 루틴 (Best Practice)

협업 시 충돌을 최소화하고 안전하게 개발하는 정석 코스입니다.

1.  **확인 (`Fetch`):** 원격 저장소의 업데이트(파란 화살표 `↓`) 확인
2.  **보관 (`Stash`):** 내가 하던 작업 `Include untracked` 체크해서 바구니에 넣기
3.  **동기화 (`Pull`):** 최신 소스 가져오기
4.  **복구 (`Stash Pop`):** 내 작업을 다시 꺼내서 최신 소스 위에 얹기
5.  **해결 (`Merge Tool`):** 부딪히는 부분(Conflict)이 있다면 눈으로 보며 합치기

---

**마무리하며:**
Git은 단순히 소스를 올리는 도구가 아니라, 내 작업 공간의 원자성을 지키는 도구입니다. 특히 `Hard Reset`과 `Stash`의 개념을 정확히 이해한다면 어떤 복잡한 협업 환경에서도 자신 있게 코딩할 수 있습니다.
