# [Git] 이클립스 'Replace With'와 'Compare With' 완벽 정리 (파일 복구 & 비교)

코딩을 하다 보면 "아, 이 파일은 그냥 원본으로 싹 다 되돌리고 싶다"거나 "내가 방금 수정한 게 기존 코드랑 어떻게 다르지?" 하고 확인해야 할 때가 있습니다.

이클립스에서 파일을 우클릭하면 나오는 **Replace With(덮어쓰기)**와 **Compare With(비교하기)**의 핵심 옵션 3가지를 정리했습니다.

---

## 1. Replace With (파일 내용 강제 원복하기)

`Replace With`는 현재 작업 중인 파일의 내용을 **과거의 특정 시점 파일로 완전히 덮어씌워 버리는(복구하는)** 기능입니다. 로컬에서 이것저것 테스트하다가 코드가 망가졌을 때 유용합니다.

| 옵션 | 기준점 | 기능 설명 (이 시점의 코드로 덮어씀) | 실무 용도 |
| :--- | :--- | :--- | :--- |
| **HEAD Revision** | 마지막 커밋 | **마지막으로 커밋(Commit)했던 상태**로 되돌립니다. | 방금 수정한 코드가 맘에 안 들어서 싹 다 지우고, 제일 최근 커밋 상태로 깔끔하게 초기화할 때 씁니다. (가장 많이 씀) |
| **Index** | 스테이징 영역 | **Add to Index(스테이징) 했던 상태**로 되돌립니다. | 파일을 수정한 뒤 `Add`를 해두고 또 수정했는데, 두 번째 수정이 맘에 안 들어서 `Add` 시점으로 돌아갈 때 씁니다. |
| **Previous Revision** | 이전 커밋 | **마지막 커밋의 바로 전 커밋(HEAD^) 상태**로 되돌립니다. | 마지막 커밋 자체의 코드가 잘못되었다고 판단해서, 그보다 한 단계 더 과거의 코드로 아예 덮어쓸 때 사용합니다. |

> **⚠️ 주의사항:** `Replace With`를 실행하면 현재 내 화면에 있던 수정 내용들은 **휴지통에도 남지 않고 영구 삭제**되므로 매우 주의해서 사용해야 합니다. (불안하면 Stash를 먼저 하세요!)

---

## 2. Compare With (변경점 비교하기)

`Compare With`는 덮어쓰지 않고, **현재 내 파일과 과거의 파일을 나란히 띄워놓고 어디가 어떻게 바뀌었는지(Diff) 비교**만 하는 기능입니다.

| 옵션 | 기준점 | 기능 설명 (현재 코드 vs 과거 코드 비교) | 실무 용도 |
| :--- | :--- | :--- | :--- |
| **HEAD Revision** | 마지막 커밋 | **마지막 커밋 이후** 내가 무엇을 추가하고 지웠는지 보여줍니다. | 커밋하기 직전에 "내가 이 파일에서 뭘 바꿨더라? 오타는 없나?" 하고 최종 점검할 때 씁니다. |
| **Index** | 스테이징 영역 | **Add 한 이후**에 추가로 수정한 내역을 보여줍니다. | `Add`를 한 번 해놓고 며칠 뒤에 다시 작업할 때, 그사이에 변경된 부분만 짧게 확인할 때 씁니다. |
| **Previous Revision** | 이전 커밋 | **이전 커밋 상태**와 현재 상태를 비교합니다. | 이번 커밋뿐만 아니라, 저번 커밋 시점과 비교해서 코드가 얼마나 달라졌는지 큰 흐름을 볼 때 사용합니다. |
| **(추가) Branch, Tag...** | 다른 브랜치 | 현재 파일과 **다른 브랜치의 동일한 파일**을 비교합니다. | 내 브랜치와 `main` 브랜치의 코드가 어떻게 다른지 확인할 때 아주 유용합니다. |

---

## 💡 실무 고수의 꿀팁 총정리

* **"아, 이 파일 하나만 어제 커밋했던 상태로 원복하고 싶어!"**
  👉 파일 우클릭 > `Replace With` > `HEAD Revision`
* **"커밋하기 전에 내가 디버깅용으로 쳐둔 콘솔 로그가 남아있나 확인해야지."**
  👉 파일 우클릭 > `Compare With` > `HEAD Revision` (비교 창에서 꼼꼼히 확인 후 지우기)

**요약하자면:**
* **Replace With**는 '타임머신 탑승 (강제 원복)'
* **Compare With**는 'CCTV 확인 (차이점 비교)'

이 두 가지만 잘 활용해도 코드가 꼬였을 때 파일 단위로 쉽고 안전하게 복구하고 검증할 수 있습니다!
